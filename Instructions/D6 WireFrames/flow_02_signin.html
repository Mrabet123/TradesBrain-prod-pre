ALL SIGN IN FLOW RULES — D2 LOCKED (10 SCREENS · 3 METHODS · 6 ERROR STATES)
APP LAUNCH & SESSION
Active session found → skip to Home (user never sees S2/S3)
No session → Welcome screen (S2)
Session check is instant (Supabase JWT validation)
Session stored in Expo SecureStore (never AsyncStorage)
THREE SIGN-IN METHODS
Method 1: Email + Password → Supabase Auth
Method 2: Google OAuth → native sheet → Supabase OAuth
Method 3: Phone + OTP (SMS) → Supabase Auth
All three shown equally on S3 — no method hidden or de-emphasised
Google OAuth cancelled → silent return to S3
SAVE PASSWORD
Toggle OFF by default — never auto-enabled
When enabled: credentials stored in Expo SecureStore
Expo SecureStore = iOS Keychain / Android Keystore
Never stored in plain text or AsyncStorage
Pre-populates fields on next sign-in when enabled
POST SIGN-IN ROUTING
Active subscription → Home · full access
Trial with queries remaining → Home · trial banner
Expired / no subscription → Home · features disabled · paywall on tap
History / Settings always accessible regardless of subscription
Routing happens silently — no loading screen shown to worker
OTP RULES (PHONE METHOD)
Code valid for 10 minutes
3 wrong codes → 5-minute lockout with countdown
Resend available after 60-second countdown
Lockout shows countdown timer — no manual unlock
"Use different number" → returns to S7
ERROR STATES
Wrong password → red border + "Incorrect password" + reset link
Account not found → "No account found" + Create Account link
5 failed attempts → 15-min lockout + countdown + reset option
Account suspended → contact support message
No internet → form data preserved + retry on reconnect
Phone not registered → "No account found" + Create Account link
